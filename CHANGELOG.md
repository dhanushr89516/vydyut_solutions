# Changelog

## How to Use This File
When creating a PR or branch:
1. Under the appropriate branch section, list all changes
2. Use the categories (Added, Changed, Fixed, etc.)
3. Keep descriptions clear and concise
4. Include file paths for easy reference

- **Added** - New features or components
- **Changed** - Changes to existing functionality
- **Fixed** - Bug fixes
- **Removed** - Removed features or code
- **Deprecated** - Features that will be removed
